# Backtrader Strategy Automation

This guide explains how to install, run, customize, and review results from the Backtrader Strategy Automation Suite.

## Overview

The suite runs every Backtrader strategy in the `strategies/` folder against one or more market symbols, saves performance metrics, creates custom Matplotlib plot files, and builds one HTML dashboard per asset.

It does not use Backtrader's built-in plotting during automation. All plots are generated by the automation runner and saved as PNG files.

## Quick Start

Run all strategies for one asset:

```bash
python run_backtest.py --symbol AAPL --period 2y --interval 1d --benchmark SPY
```

Build the asset dashboard:

```bash
python build_dashboard.py --results results --mode assets
```

Open the dashboard inside the asset folder, for example:

```text
results/AAPL/dashboard.html
```

## Running One Symbol

Use `run_backtest.py` to run every strategy in `strategies/` for a single symbol.

Example:

```bash
python run_backtest.py --symbol MSFT --period 2y --interval 1d --benchmark SPY
```

Crypto example:

```bash
python run_backtest.py --symbol BTC-USD --start 2022-01-01 --end 2025-01-01 --interval 1d --benchmark SPY
```

## Command Options

`run_backtest.py` supports these options:

```text
--symbol       Required. Ticker symbol to test, such as AAPL, SPY, BTC-USD.
--period       yfinance period, such as 6mo, 1y, 2y, 5y.
--start        Start date, such as 2022-01-01.
--end          End date, such as 2025-01-01.
--interval     Data interval, such as 1d, 1h, 30m.
--benchmark    Benchmark ticker. Default is SPY.
--strategies   Strategy folder. Default is strategies.
--cash         Starting cash. Default is 10000.
--commission   Broker commission rate. Default is 0.001.
--stake-percent Percent of available cash used by the default sizer. Default is 95.
--risk-free-rate Annual risk-free rate used for Sharpe. Default is 0.0.
--out          Output folder. Default is results.
--allow-short  Allow strategies to open short positions. Default is long-only.
--strict       Stop immediately if a strategy errors.
--verbose      Show strategy print/log output.
```

Use either `--period` or `--start` plus `--end`.

## Batch Runs

The included batch launchers run multiple symbols and then build a dashboard inside each asset's result folder.

### Windows

```bat
run_all.bat
```

### macOS/Linux

```bash
bash run_all.sh
```

You can edit the asset list inside these files:

```text
AAPL MSFT TSLA SPY
```

You can also change:

```text
PERIOD=2y
INTERVAL=1d
BENCHMARK=SPY
STAKE_PERCENT=95
```

`STAKE_PERCENT` is important for high-priced assets such as `BTC-USD`; it lets the system buy fractional position sizes instead of trying to buy one full coin/share.

Each strategy gets a `metrics.csv` file containing:

```text

```

Each symbol also gets `all_metrics.csv`, which combines all strategy metrics for that symbol.

Sharpe is calculated from the saved strategy equity curve, annualized from the actual date spacing in `equity.csv`. This avoids misleading values from Backtrader's built-in Sharpe analyzer when crypto trades daily, when strategies sit flat for long periods, or when return streams have near-zero volatility.

## Dashboards

Build asset dashboards after one or more backtests:

```bash
python build_dashboard.py --results results --mode assets
```

This creates:

```text
results/AAPL/dashboard.html
results/MSFT/dashboard.html
results/TSLA/dashboard.html
```

Each asset dashboard includes:

- Summary metrics table
- Return comparison chart
- Drawdown comparison chart
- Sharpe ratio chart
- Embedded saved Matplotlib plots for each strategy

To create one combined dashboard across all assets:

```bash
python build_dashboard.py --results results --mode combined --out dashboard.html
```

## Adding A New Strategy

Add a new `.py` file inside `strategies/`.

Each strategy file must contain at least one Backtrader strategy class:

```python
import backtrader as bt


class MyStrategy(bt.Strategy):
    params = dict(period=20)

    def __init__(self):
        self.sma = bt.ind.SMA(self.data.close, period=self.p.period)

    def next(self):
        if not self.position and self.data.close[0] > self.sma[0]:
            self.buy()
        elif self.position and self.data.close[0] < self.sma[0]:
            self.sell()
```

The automation will automatically discover strategy classes in the file.

## Strategy Compatibility Rules

For best results, strategy files should:

- Define normal `bt.Strategy` classes.
- Avoid downloading data inside the strategy file.
- Avoid calling `cerebro.run()` inside the strategy file.
- Avoid calling `cerebro.plot()` or `plt.show()`.
- Avoid requiring custom local model files unless they are included with the product.
- Work with a standard single OHLCV data feed.

The runner protects against popup plotting, but clean strategy files are easier to maintain and sell.

## Quiet And Verbose Modes

By default, strategy print output is suppressed so batch runs stay clean.

To see strategy logs:

```bash
python run_backtest.py --symbol AAPL --period 1y --verbose
```

## Handling Failed Strategies

If a strategy fails, the runner continues by default and records the problem in:

```text
results/SYMBOL/skipped_strategies.csv
```

Use `--strict` if you want the run to stop at the first error:

```bash
python run_backtest.py --symbol AAPL --period 1y --strict
```

## Common Examples

Run AAPL for two years:

```bash
python run_backtest.py --symbol AAPL --period 2y --interval 1d --benchmark SPY
```

Run SPY for five years:

```bash
python run_backtest.py --symbol SPY --period 5y --interval 1d --benchmark QQQ
```

Run Bitcoin over a fixed date range:

```bash
python run_backtest.py --symbol BTC-USD --start 2022-01-01 --end 2025-01-01 --interval 1d --benchmark SPY
```

Use different starting cash:

```bash
python run_backtest.py --symbol AAPL --period 2y --cash 50000
```

Run BTC-USD with fractional sizing:

```bash
python run_backtest.py --symbol BTC-USD --period 2y --benchmark BTC-USD --stake-percent 95
```

Allow short entries only when you intentionally want long/short behavior:

```bash
python run_backtest.py --symbol BTC-USD --period 2y --allow-short
```

Use a lower commission:

```bash
python run_backtest.py --symbol AAPL --period 2y --commission 0.0005
```

## Troubleshooting

### No data downloaded

Check the ticker symbol, date range, interval, and internet connection.

Example:

```bash
python run_backtest.py --symbol AAPL --period 1y --interval 1d
```

### Dashboard says no metrics found

Run at least one backtest first:

```bash
python run_backtest.py --symbol AAPL --period 1y
python build_dashboard.py --results results --mode assets
```

### A strategy was skipped

Open:

```text
results/SYMBOL/skipped_strategies.csv
```

This file shows which strategy failed and at which stage.

### Too much output

Do not use `--verbose`. The default mode suppresses strategy logs.

### I want to inspect strategy logs

Use:

```bash
python run_backtest.py --symbol AAPL --period 1y --verbose
```
